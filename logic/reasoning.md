# 🧠 設計推理引擎 (Design Reasoning Engine)

## 2. 決策邏輯 (Decision Logic)

### 用戶意圖分析 (User Intent Analysis)
分析用戶的需求以決定：
- **產業限制**：(例如：醫療需要乾淨/信任感；遊戲需要深色/霓虹)
- **內容密度**：(例如：儀表板 = 高密度；Landing Page = 低/中密度)
- **語調**：(例如：活潑、嚴肅、奢華)

### 風格映射 (Style Mapping - 自動選擇)
根據意圖從 `ui/styles/index.md` 選擇最適合的風格：
- **科技/SaaS**：-> 玻璃擬態 (Glassmorphism)、新粗獷主義 (Neo-Brutalism)、簡潔商務
- **奢華/時尚**：-> 雜誌風格 (Editorial)、極簡主義 (Minimalist)、暗黑奢華
- **福利/健康**：-> 柔和 UI (Soft UI)、自然風 (Natural)、北歐風 (Scandi)
- **創意/設計**：-> 粗獷主義 (Brutalism)、全像術 (Holographic)、Y2K

### 反模式過濾器 (Anti-Pattern Filter)
主動拒絕通用或不適當的選擇：
- ❌ **通用感**：Bootstrap 預設藍色、庫存照片、「Lorem Ipsum」假文
- ❌ **衝突感**：律師事務所使用霓虹色、科技新創使用 Times New Roman

## 3. 執行觸發 (Execution Trigger)
輸出表格後，自動提議基於這些選擇生成 **設計系統主檔** (`design-system/MASTER.md`)。
