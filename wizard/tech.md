## 階段 1/6：專案基本資訊

### Q1. 專案類型

請問您要建立什麼類型的網站/應用？

| 選項 | 說明 | 技術傾向 |
|-----|------|---------|
| A) 🏢 形象網站 | 公司介紹、品牌展示 | 靜態/SSG |
| B) 📰 內容網站 | 部落格、新聞、雜誌 | SSG/SSR + CMS |
| C) 🛒 電商網站 | 商品販售、購物車 | SSR + 完整後端 |
| D) 🖥️ 網頁應用 | SaaS、工具、平台 | SPA/SSR + API |
| E) 📊 管理後台 | Dashboard、CMS | SPA + API |
| F) 📱 行動應用 | PWA、行動優先 | PWA 框架 |
| G) 🎮 互動體驗 | 遊戲、動畫、3D | WebGL/Canvas |
| H) 📋 其他 | 請描述需求 | 依需求判斷 |

**判斷邏輯：**
```
A/B → 優先考慮 SSG（Next.js、Nuxt.js、Astro）
C → 需要 SSR + 完整後端（Next.js + API 或 Laravel）
D/E → SPA 可行（React、Vue）或 SSR
F → PWA 能力（Next.js、Nuxt.js）
G → 特殊需求，依詳情判斷
```

## 階段 2/6：規模與複雜度

### Q2. 網站規模

預估的網站規模？

| 選項 | 頁面數 | 建議 |
|-----|-------|-----|
| A) 🟢 微型 | 1-3 頁 | 可用 Vanilla 或輕量框架 |
| B) 🔵 小型 | 3-10 頁 | 任何框架皆可 |
| C) 🟡 中型 | 10-30 頁 | 建議使用主流框架 |
| D) 🟠 大型 | 30-100 頁 | 需要良好架構規劃 |
| E) 🔴 超大型 | 100+ 頁 | 考慮微前端/模組化 |

### Q3. 功能複雜度

| 選項 | 說明 | 後端需求 |
|-----|------|---------|
| A) 簡單 | 純展示，無互動 | 可無後端 |
| B) 基本 | 表單、搜尋 | 輕量後端/Serverless |
| C) 中等 | 會員系統、資料庫 | 完整後端 |
| D) 複雜 | 即時通訊、複雜邏輯 | 完整後端 + WebSocket |
| E) 高度複雜 | 多租戶、微服務 | 企業架構 |

**判斷邏輯：**
```
規模 A/B + 複雜度 A/B → 可考慮輕量方案
規模 C+ 或 複雜度 C+ → 建議主流框架
規模 E 或 複雜度 E → 需要進階架構規劃
```

## 階段 3/6：效能與 SEO 需求

### Q4. SEO 重要性

| 選項 | 說明 | 渲染策略 |
|-----|------|---------|
| A) 🔴 非常重要 | 需要搜尋排名 | SSR/SSG 必須 |
| B) 🟡 中等重要 | 部分頁面需要 | 混合渲染 |
| C) 🟢 不太重要 | 內部系統/需登入 | SPA 可行 |

**判斷邏輯：**
```
A → 必須 SSR/SSG：Next.js、Nuxt.js、Laravel + Blade
B → 混合方案：Next.js App Router、Nuxt.js Hybrid
C → SPA 可行：React SPA、Vue SPA
```

### Q5. 首次載入速度需求

| 選項 | 時間 | 建議 |
|-----|------|-----|
| A) 極快 | < 1秒 | SSG + CDN + 極致優化 |
| B) 快速 | 1-2秒 | SSR 或 SSG |
| C) 普通 | 2-3秒 | 一般 SPA 可接受 |
| D) 無特別要求 | - | 依其他因素決定 |

## 階段 4/6：團隊與技術背景

### Q6. 技術背景

| 選項 | 說明 | 建議學習曲線 |
|-----|------|------------|
| A) 😊 前端為主 | 熟悉 HTML/CSS/JS | 可嘗試 React/Vue |
| B) 💻 全端開發 | 前後端都熟 | 任何方案皆可 |
| C) 🖥️ 後端為主 | 熟悉 PHP/Python/Node | 選擇熟悉的後端 + 學前端 |
| D) 🆕 新手 | 剛開始學習 | 選擇文件完善、社群活躍的 |
| E) 🎨 設計師轉開發 | 熟悉設計工具 | 可考慮 Low-code 或 Vue |

### Q7. 最熟悉的框架（可複選）

**前端：**
- [ ] React
- [ ] Vue
- [ ] Angular
- [ ] Svelte
- [ ] Vanilla JS / jQuery
- [ ] 都不熟，願意學習

**後端：**
- [ ] PHP / Laravel
- [ ] Node.js / Express
- [ ] Python / Django
- [ ] Python / FastAPI
- [ ] Ruby on Rails
- [ ] .NET
- [ ] 都不熟，願意學習

**判斷邏輯：**
```
有熟悉的 → 優先推薦熟悉的（學習成本低）
都不熟 → 根據專案需求推薦最適合的
新手 → 推薦 Vue（學習曲線較平緩）或 Laravel（文件完善）
```

| 選項 | 人數 | 考量 |

| B) 👥 小團隊 | 2-5 人 | 需要基本規範 |
| C) 👨‍👩‍👧‍👦 中型團隊 | 5-15 人 | 需要清晰架構 |
| D) 🏢 大型團隊 | 15+ 人 | 考慮模組化、微前端 |

## 分析與推薦邏輯

### 權重計算

| 因素 | 權重 |
|-----|------|
| 團隊熟悉度 | 30% |
| SEO 需求 | 25% |
| 專案複雜度 | 20% |
| 開發時程 | 15% |
| 特殊需求 | 10% |

### 推薦決策樹

```
SEO 重要？
├─ 是 → SSR/SSG 必須
│   ├─ 熟悉 React → Next.js
│   ├─ 熟悉 Vue → Nuxt.js
│   └─ 熟悉 PHP → Laravel + Inertia
│
└─ 否 → SPA 可行
    ├─ 熟悉 React → React + Vite
    ├─ 熟悉 Vue → Vue + Vite
    └─ 後端為主
        ├─ PHP → Laravel + Livewire
        ├─ Python → Django / FastAPI + 前端
        └─ Node → Express + 前端
```
